# Grouping Text
HTML 요소들을 개별적으로 다루다 보면 여러 요소를 하나의 단위로 묶어서 스타일을 적용하거나 구조를 잡아야 하는 상황이 생깁니다. 이때 사용하는 것이 `<div>`와 `<span>` 입니다. 둘 다 자체적인 시각적 의미는 없으며, **CSS나 JavaScript와 함께 사용할 때 비로소 의미가 생깁니다.**

---
## `<div>`
**블록 레벨(Block-level)** 요소이자 흐름 콘텐츠를 담는 범용 컨테이너입니다. 항상 새로운 줄에서 시작하며 가로 전체 너비를 차지합니다.

```html
<!-- CSS 적용 전 -> 화면에 아무 변화 없음 -->
<div>
	<p>제목</p>
	<p>본문 내용</p>
</div>

<!-- CSS 적용 후 -> 비로소 의미가 생김 -->
<div style="background-color: gray; padding: 20px;">
	<p>제목</p>
	<p>본문 내용</p>
</div>

<!-- 부모에 Flexbox 적용 시 자식을 묶는 단위로 활용 -->
<div style="display: flex;">
	<div>왼쪽</div>
	<div>오른쪽</div>
</div>
```

---
## `<span>`
**인라인(Inline)** 요소이자 텍스트 일부를 묶는 범용 컨테이너입니다. 새로운 줄을 만들지 않고 텍스트 흐름 안에서 동작합니다. `<div>`와 마찬가지로 그 자체로는 아무 의미가 없으며, CSS나 JavaScript로 텍스트의 특정 부분을 타겟팅할 때 유용합니다. `lang`, `dir` 같은 공통 속성을 공유하는 요소를 묶는 용도로도 사용합니다.

```html
<!-- 텍스트 일부에만 스타일 적용 -->
<p>이 문장에서 <span style="color: red;">이 부분만</span> 빨간색입니다.</p>

<!-- JavaScript로 특정 텍스트 타겟팅 -->
<p>상태: <span id="status">대기 중</span></p>
```

---
# 정리
## `<div>`
- 요소 유형: 블록 레벨
- 줄바꿈: 자동으로 발생
- 주요 용도: 레이아웃 구조 그룹핑
## `<span>`
- 요소 유형: 인라인
- 줄바꿈: 발생하지 않음
- 주요 용도: 텍스트 일부 스타일링 / 타겟팅
